Part A provided more precise control over the layout and styling with smooth animations, hover effects, and the responsive layout for different screen sizes. The idea was to try and make the page seem creatively unique. But on the other hand, bootstrap/Part B made the dev process easier with components and responsive layouts already. From a point of view of speed, it’s what I used to make the landing page as well to just quickly put something together and submit. The downside is that it might be less unique, but you can of course add on more styling to make it nicer.

Some challenges were having to reorganize the html to play nice with part B, the default fonts of bootstrap etc. I’ve left them as is as I couldn’t be bothered. There are definitely optimizations and redundancies that are left over in the CSS, but it is what it is; the functionality is there. I personally enjoyed making things from scratch with the pure css approach as I felt like I had more control over things. Realistically I would see myself using a melange of libraries and custom CSS to build on whatever I might want to add/override depending on what the project requires.
